使用实验性的产物驱动方法开始新的变更。

**输入**：用户的请求应包含变更名称（短横线分隔）或者对他们想要构建内容的描述。

**步骤**

1. **如果没有提供清晰的输入，询问他们想要构建什么**

   使用**AskUserQuestion 工具**（开放式，无预设选项）来询问：
   > "你想要进行什么变更？描述你想要构建或修复的内容。"

   从他们的描述中导出短横线分隔的名称（例如，"添加用户认证" → `add-user-auth`）。

   **重要**：在没有理解用户想要构建什么之前不要继续。

2. **创建变更目录**
   ```bash
   openspec new change "<name>"
   ```
   这会在 `openspec/changes/<name>/` 创建一个已搭建脚手架的变更。

3. **显示产物状态**
   ```bash
   openspec status --change "<name>"
   ```
   这显示需要创建哪些产物以及哪些已准备就绪（依赖已满足）。

4. **获取第一个产物的指令**
   第一个产物始终是 `proposal`（无依赖）。
   ```bash
   openspec instructions proposal --change "<name>"
   ```
   这会输出创建提案的模板和上下文。

5. **停止并等待用户指示**

**输出**

完成步骤后，总结：
- 变更名称和位置
- 当前状态（0/4 个产物完成）
- 提案产物的模板
- 提示："准备好创建提案了吗？只需描述这个变更的内容，我会起草提案，或者让我继续。"

**护栏**
- 还不要创建任何产物——只显示指令
- 不要超越显示提案模板
- 如果名称无效（不是短横线分隔），请求有效的名称
- 如果具有该名称的变更已存在，建议继续该变更
